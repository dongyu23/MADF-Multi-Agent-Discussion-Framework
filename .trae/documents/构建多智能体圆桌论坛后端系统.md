# 毕业设计：圆桌论坛多智能体后端系统设计方案

本方案旨在实现一个基于大模型（GLM-4-Air）的多智能体圆桌论坛后端。系统包含上帝视角的角色生成、主持人控场、以及具备"快思考"与"慢发言"机制的参会嘉宾智能体。

## 1. 系统架构与环境准备

### 环境配置
- **虚拟环境**: 创建并使用 python 虚拟环境 `bysj`。
- **依赖库**: 安装 `zhipuai` 用于调用 GLM 模型。
- **配置文件**: `config.py` 存储 API Key 和模型名称常量。

### 核心模块结构
- `main.py`: 主程序/编排器（Orchestrator），负责循环调度、时间管理、优先级控制。
- `god.py`: "上帝"角色，负责根据主题生成差异化的角色配置（Persona）。
- `agent.py`: 定义 `ModeratorAgent`（主持人）和 `ParticipantAgent`（嘉宾）。
- `memory.py`: 定义 `SharedMemory`（共享记忆）和 `PrivateMemory`（私有记忆）。
- `utils.py`: 辅助函数（如 GLM 调用封装、JSON 解析）。

## 2. 详细模块设计

### 2.1 记忆模块 (`memory.py`)
- **SharedMemory**:
  - `message_window`: 消息滑动窗口，上限为 $n$（参会人数）。存储 `{speaker, content}`。
  - `summary_history`: 总结消息窗口。存储主持人生成的阶段性总结。
  - 功能：添加消息、检查是否满 $n$ 条、清空窗口、添加总结。
- **PrivateMemory**:
  - `thought_window`: 存储智能体自己的思考历史，上限为 $n$。

### 2.2 角色生成 (`god.py`)
- 功能：接收主题和人数 $n$。
- 逻辑：调用 LLM 生成 $n$ 个具有不同 MBTI、领域背景、持有不同立场的角色设定（System Prompt）。

### 2.3 智能体类 (`agent.py`)
- **BaseAgent**: 基础属性（名字、设定）。
- **ModeratorAgent**:
  - `opening()`: 开场白，介绍背景和嘉宾。
  - `periodic_summary(messages)`: 对滑动窗口内的消息进行总结。
  - `closing(summary_history)`: 最终总结，结束论坛。
- **ParticipantAgent**:
  - 属性：`priority` (初始 100), `private_memory`.
  - `think(context)`: "快思考"模型。
    - 输入：当前论点、历史摘要。
    - 输出：JSON `{focus, attitude, processing, action}`。
  - `speak(thought, context)`: 正式发言。
    - 输入：自己的思考结果、当前上下文。
    - 输出：流式文本发言。

### 2.4 编排器逻辑 (`main.py`)
系统的核心循环，模拟时间流逝和事件触发。

1. **初始化**:
   - 用户输入：`theme`, `n_participants`, `duration_minutes`.
   - 调用 God 生成 $n$ 个角色。
   - 初始化主持人。
   - 记录 `start_time`。

2. **开场**: 主持人发言。

3. **主循环**:
   - **时间检查**: `if (now - start) > duration`: 主持人总结 -> 退出。
   - **总结检查**: `if len(message_window) >= n`: 主持人总结 -> 清空窗口 -> 存入总结历史。
   - **发言权竞争**:
     - 筛选当前优先级最高的智能体集合。
     - 并发（或顺序模拟）执行 `think()`。
     - 检查谁输出了 `apply_to_speak`。
     - **调度策略**: 响应第一个申请的智能体（或在同优先级中随机）。
   - **执行发言**:
     - 选中者执行 `speak()`。
     - 选中者 `priority -= 1`。
     - 消息存入 `SharedMemory`。
     - 未选中但思考过的智能体，将其思考存入 `PrivateMemory`。

## 3. 实施步骤

1. **基础建设**: 创建文件结构，配置 `zhipuai` 客户端。
2. **记忆与角色**: 实现 Memory 类和 God 生成逻辑。
3. **智能体核心**: 实现主持人功能和嘉宾的 Think/Speak 流程（含 Prompt 设计）。
4. **主控流程**: 编写 `main.py` 串联逻辑，实现优先级和时间控制。
5. **测试与验证**: 模拟运行，验证 JSON 输出解析、优先级变更和总结触发机制。
